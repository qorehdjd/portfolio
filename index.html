성장하는 프론트개발자 남궁혁입니다.

    개발하면서 어려웠던 점은?

  • 1. 빠르게 전송 버튼을 연속으로 누르면 서버쪽으로 여러개의 요청이 가는 문제가 발생했습니다.
    => useRef에 값을 0을주고 useRef가 0일때만 코드가 실행될 수 있게 분기처리해서 구현했습니다. 만일 유저가 서버단으로 요청 버튼을 누르게되면 useRef를 1로 바꿔서 연속으로 버튼을 요청해도 서버쪽으로 요청이 가지않게 코드를 수정했습니다.그리고 응답을 받으면 라이프사이클인 useEffect를 이용하여 useRef를 0으로 초기화해서 다시 버튼이 동작하게 만들었습니다.
  • 2. cors문제로 인하여 서버로 요청을해도 응답이 없었습니다.
    => sop정책으로 인하여 같은 origin일때만 서로 주고 받을 수 있다는 것을 깨닫고 서버단에서 cors라이브러리를 이용하여 프론트 url을 적어줬습니다. 그리고 서버단에서는 제대로 응답을 해주지만 브라우저가 그 응답을 막는것이므로 또 다른 방법으로는 froxy서버를 둬서 해결할 수 있다는 것을 알았습니다.
  • 3. 로컬 환경에서는 문제가 없던 쿠키전송이 배포후에 문제가 발생했습니다.
    => 로컬 환경에서는 localhost로 같은 도메인이였기 때문에 쿠키 전송에 문제가 없었지만, 배포후에는 서로 다른 도메인을 가지고 있었기 때문에 쿠키 전송이 안된다는 것을 알았습니다. 그래서 도메인을 구입하였고 프론트에는 도메인주소를 백엔드에서는 서브도메인을 이용하여 문제를 해결했습니다.
  • 4. 페이지를 새로고침하면 redux store에 저장된 state들이 초기화 되는 문제가 발생했습니다.
    => 이 문제는 총 2가지 해결방법이 있었는데 첫번째는 서버사이드렌더링을 이용하여 새로고침 후에도 먼저 렌더링된 페이지를 가져와서 데이터를 유지하는 방법이였고 두번째는 redux-persist를 이용하여 로컬스토리지,세션스토리지에 저장하는 것이 였습니다. 저는 후자인 redux-persist를 선택하였고 로컬스토리지와 세션스토리지중에서 사용자가 페이지를 닫으면 정보가 초기화 될 수 있도록 세션스토리지를 선택하여 구현했습니다.
  • 5. useEffect가 처음 렌더링되고 난 후에 실행되는데 이것을 때문에 오류가 발생했습니다.
    => 가령 post를 작성한 유저가 작성을 완료하고 나면 useEffect를 이용하여 redux state addPostDone을 이용한 분기처리(useEffect(()=> { if(addPostDone) { router.push('/index')}}, []))로 index페이지로 보낸다고 했을 때, 또 다시 유저가 post를 작성하려고하면 useEffcet가 실행돼서 index페이지로 가게되는 에러가 발생하게 되었습니다.(addPostDone이 이미 true이기때문) 이 문제를 useRef를 이용하여 해결하였는데 useRef에다가 false값을 주고 첫 렌더링 이후에 useRef값을 true로 바꿔줘서 router.push('/index')가 useRef값이 true일때만 실행될 수 있게 분기처리하여 해결하였습니다.

기술스택

  • HTML

    시맨틱 마크업과 브라우저 렌더링 프로세스에 대해서 잘 알고 있습니다.

  • CSS

    block과 inline의 차이를 잘 알고 flex를 이용하여 복잡한 레이아웃도 구현할 수 있습니다.

  • JAVASCRIPT

    자바스크립트의 비동기 프로세스, 실행 컨텍스트, this 바인딩, Promise, Prototype등과 같은 코어 개념들을 잘 이해하고 있습니다.

  • REACT

    Virtual DOM, life cycle, props, 컴포넌트의 리렌더링조건등을 잘 이해하고 있습니다.

  • NEXT.JS

    NEXT.JS의 서버사이드렌더링을 이용하여 SEO문제를 해결할 수 있습니다.

  • REDUX/REDUX-TOOLKIT

    효율적인 컴포넌트 상태관리 방법과 Redux-Thunk, Redux-Saga와 같은 미들웨어를 잘 다룹니다.

  • STYLED-COMPONENT

    CSS-in-JS인 styled-component를 이용하여 js파일안에 스타일을 작성하여 컴포넌트화 시킬수 있습니다.

  • TYPESCRIPT

    TYPESCRIPT를 이용하여 버그가 적은 코드를 작성할 수 있습니다.

  • NODE.JS

    EXPRESS를 이용하여 CRUD, search, passport와 session을 이용한 로그인을 구현할 수 있습니다.

  • MONGO_DB

    mongoose를 이용하여 create(생성), read(읽기), update(갱신), delete(삭제), 로그인, 로그아웃, 정규표현식을 이용한 search등을 구현할 수 있습니다.

포트폴리오

홈페이지 접속하기

사용자가 페이지에 들어왔을 때 보이는 첫 화면 index 페이지입니다.
첫 페이지를 구성할 때 중요하게 생각한 것은 SEO 최적화 작업이었습니다.
next.js의 getServersideProps를 이용하여 렌더링 된 페이지를 보여줘서 검색엔진 동작에 적합하게 맞추어 웹사이트 노출이 더 잘 되게 만들었습니다.
그리고 새로고침을 하게 되면 로그인 한 유저가 로그아웃이 된 것처럼 표시가 되는데 이것은 redux store의 state가 초기화되는 문제로 redux-persist를 이용하여 세션 스토리지에 유저 정보를 저장하는 로직을 구현했습니다.

서버로부터 데이터를 다 가져오게 되면 사용자 입장에서는 너무 많은 시간을 기다리게 되면서 ux 경험이 떨어지므로 post를 20개씩 나눠서 페이지 네이션을 구현했습니다.
그리고 제목, 내용, 제목+내용, 글쓴이로 category를 구분해서 serach 기능을 구현했습니다.

서버로부터 받은 데이터를 post 페이지에 뿌려줘서 component 단위로 post를 구분했습니다.

post 페이지로 추천, 댓글, 대댓글, 수정, 삭제, 글쓰기를 구현했습니다.
이러한 기능들은 로그인한 유저만 이용할 수 있게 구현했고 또한 수정, 삭제 같은 경우는 post를 작성할 때 password를 넣어야 되는데 그 password랑 일치해야 수정, 삭제를 할 수 있게 구현했습니다.
그리고 댓글, 대댓글에서 댓글은 삭제하면 대댓글들 모두가 같이 삭제가 되고 대댓글은 삭제하면 대댓글만 삭제되게 구현했습니다.
댓글, 대댓글 또한 작성할 때 password를 넣어야 되는데 이 password랑 일치해야 삭제가 가능합니다.

로그인페이지로 아이디와 비밀번호가 일치하지 않을 경우 에러 메세지를 하단에서 보여주게 됩니다.
